# 🏗️ 1. System Architecture

```
            ┌──────────────┐
            │   User Query │
            └──────┬───────┘
                   ↓
        ┌────────────────────┐
        │  Retrieval Layer   │
        │ (ChromaDB Search)  │
        └──────┬─────────────┘
               ↓
        ┌────────────────────┐
        │   Context Builder  │
        └──────┬─────────────┘
               ↓
        ┌────────────────────┐
        │  Local LLM (Ollama)│
        └──────┬─────────────┘
               ↓
            Answer
```

### Pipeline:

1. **Ingestion pipeline** → process docs → embeddings → store in Chroma
2. **Retrieval pipeline** → query → similarity search → LLM generation

# ⚙️ 4. Ingestion Pipeline

## Steps:

### 1. Load Data

* PDF, DOCX, TXT, HTML

```python
from langchain.document_loaders import PyPDFLoader
```

### 2. Chunking (VERY IMPORTANT)

```python
from langchain.text_splitter import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=50
)
```

👉 Chunking improves retrieval accuracy

---

### 3. Embeddings

```python
from langchain.embeddings import HuggingFaceEmbeddings

emb = HuggingFaceEmbeddings(
    model_name="all-MiniLM-L6-v2"
)
```

---

### 4. Store in ChromaDB

```python
from langchain.vectorstores import Chroma

db = Chroma.from_documents(
    documents=chunks,
    embedding=emb,
    persist_directory="./chroma_db"
)
db.persist()
```

👉 Vector DB stores text as embeddings for similarity search ([Real Python][3])

---

# 🔍 5. Retrieval Pipeline

```python
retriever = db.as_retriever(search_kwargs={"k": 5})

docs = retriever.get_relevant_documents(query)
```

---

# 🧠 6. Local LLM Setup

Install:

```bash
ollama pull mistral:7b
```

Use in Python:

```python
from langchain.llms import Ollama

llm = Ollama(model="mistral")
```

---

# 🔗 7. RAG Chain

```python
from langchain.chains import RetrievalQA

qa = RetrievalQA.from_chain_type(
    llm=llm,
    retriever=retriever
)

qa.run("your query")
```

👉 RAG = Retrieval + Generation ([LangChain Docs][4])

# 💾 8. Storage Planning (1GB Data)

### Rough estimation:

* Raw data: 1GB
* After chunking: ~3–5GB (with embeddings)

### Tips:

* Use **smaller chunks (300–500)**
* Use **light embedding model**
* Store in SSD

---

# ⚡ 9. Performance Optimization

### MUST DO:

* Use **quantized models (Q4/Q5)**
* Limit context window (2k–4k)
* Use **k=3–5 retrieval**

### OPTIONAL:

* Cache embeddings
* Multi-thread ingestion
* Use FAISS fallback if needed

# 🚫 10. Offline Constraints Handling

| Problem        | Solution            |
| -------------- | ------------------- |
| No internet    | preload models      |
| Memory limit   | quantization        |
| Slow inference | smaller models      |
| Large docs     | chunking + batching |
